# Basic Fixture

This file exercises all six L2 reference kinds.

## File references

See [auth module](src/auth.ts) for details.

Also refer to `cli/commands/docs/extract.ts` directly.

## URL references

Visit the [project homepage](https://example.com/docs) for documentation.

## CLI references

Run `oma docs verify` to check documentation drift.

Use `git status` to see unstaged changes.

## Script references

Run `bun run test` to execute the test suite.

Also `npm run lint` to check code style.

## Env var references

Set `process.env.OPENAI_API_KEY` before starting the service.

The `$DATABASE_URL` env var must point to your database.

## Config references

Enable `docs.auto_verify` in your oma-config.yaml to auto-run on workflows.

The `language` setting controls the response language.
