## 概要

`oma-config.yaml`は次の2か所に置くことができます。

- **プロジェクト**：`<cwd>/.agents/oma-config.yaml`
- **グローバル**：`~/.agents/oma-config.yaml`

両方のファイルが存在する場合、すべてのキーについてプロジェクト側のファイルが優先されます。これは意図的な動作です。プロジェクトごとのカスタマイズはより具体的なシグナルであり、ユーザー全体のデフォルトによって上書きされるべきではないためです。

## 優先順位テーブル

| キー | プロジェクト優先？ | 備考 |
|-----|:---:|-------|
| `auto_update_cli` | はい | プロジェクト値がグローバルを上書きします。`resolveAutoUpdateCli`（`cli/commands/update/update.ts`）に実装されています。 |
| `serena.mode` | はい | Serena MCPのトランスポートモードを制御します（例：`stdio`、`sse`）。 |
| `telemetry` | はい | ベンダーテレメトリのopt-in（`true` / `false`）。 |
| `language` | はい | エージェント出力の応答言語（例：`en`、`ko`、`ja`）。 |
| `model_preset` | はい | モデル選択プリセット（例：`claude`、`mixed`、`codex`）。 |
| `translation_voice` | はい | 翻訳トーン：`formal`、`balanced`、`interpreter`。 |
| `timezone` | はい | タイムゾーン識別子（例：`Asia/Seoul`、`America/New_York`）。 |

「プロジェクト優先」とは次の意味です。当該キーがプロジェクトファイルに存在する場合、グローバルファイルの内容にかかわらずプロジェクト側の値が使用されます。プロジェクトファイルに存在しなければ、グローバルファイルの値が使用されます。両方とも存在しなければデフォルト値が適用されます。

## デフォルト値

| キー | デフォルト | 適用条件 |
|-----|---------|--------------|
| `auto_update_cli` | `true` | 両ファイルが存在しないか、キーが欠落している場合 |
| `serena.mode` | `stdio` | 両ファイルが存在しないか、キーが欠落している場合 |
| `telemetry` | `false` | 両ファイルが存在しないか、キーが欠落している場合 |
| `language` | `en` | 両ファイルが存在しないか、キーが欠落している場合 |
| `model_preset` | `claude` | 両ファイルが存在しないか、キーが欠落している場合 |
| `translation_voice` | `balanced` | 両ファイルが存在しないか、キーが欠落している場合 |
| `timezone` | システムタイムゾーン | 両ファイルが存在しないか、キーが欠落している場合 |

## 読み取り順序の根拠

プロジェクト設定が先に読み込まれるのは、それがより具体的なコンテキスト（開発者が実際に作業しているリポジトリ）を表しているからです。チームがプロジェクトに対して`language: ko`や`model_preset: mixed`を強制している場合、その選択が個人のグローバル`oma-config.yaml`によって暗黙のうちに上書きされてはなりません。

グローバルファイルはユーザー全体のベースラインを提供します。プロジェクトで設定されていないキーはグローバル値へフォールバックし、グローバルにも値がなければ最終的にハードコードされたデフォルト値へフォールバックします。

## 補足

- `oma-config.yaml`の`language`はエージェントの応答言語を制御します。インストールやアップデートの警告メッセージの決定には使用され**ません**。インストール時点では`oma-config.yaml`がまだ読み込まれていないため、それらのメッセージはシステムロケール（`$LANG`）に基づきます。
- `auto_update_cli`の優先順位はupdateコマンド内で明示的に実装されています。プロジェクトインストールとグローバルインストールの両方が存在する場合、プロジェクトの`oma-config.yaml`が最初に参照されます。
- `oma-config.yaml`を直接編集しても問題ありません。`oma install`および`oma update`は正規表現レベルのフィールド置換を用いており、それらが管理対象としていないユーザー編集済みキー（例：カスタムの`agents:`オーバーライド、`session.quota_cap`）はそのまま保持されます。
