## Обзор

`oma-config.yaml` может располагаться в двух местах:

- **Проект**: `<cwd>/.agents/oma-config.yaml`
- **Глобально**: `~/.agents/oma-config.yaml`

Когда оба файла существуют, файл проекта побеждает по каждому ключу. Это сделано намеренно: настройка для отдельного проекта — более специфичный сигнал, и её не должно перекрывать значение по умолчанию на уровне пользователя.

## Таблица приоритета

| Ключ | Проект побеждает? | Примечания |
|-----|:---:|-------|
| `auto_update_cli` | Да | Значение из проекта перекрывает глобальное. Реализовано в `resolveAutoUpdateCli` (`cli/commands/update/update.ts`). |
| `serena.mode` | Да | Управляет режимом транспорта Serena MCP (например, `stdio`, `sse`). |
| `telemetry` | Да | Согласие на телеметрию вендора (`true` / `false`). |
| `language` | Да | Язык ответов для вывода агентов (например, `en`, `ko`, `ja`). |
| `model_preset` | Да | Пресет выбора модели (например, `claude`, `mixed`, `codex`). |
| `translation_voice` | Да | Тон переводчика: `formal`, `balanced`, `interpreter`. |
| `timezone` | Да | Идентификатор часового пояса (например, `Asia/Seoul`, `America/New_York`). |

«Проект побеждает» означает: если ключ присутствует в файле проекта, используется его значение независимо от того, что задано в глобальном файле. Если ключ отсутствует в файле проекта, используется значение из глобального файла. Если он отсутствует в обоих, применяется значение по умолчанию.

## Значения по умолчанию

| Ключ | По умолчанию | Когда применяется |
|-----|---------|--------------|
| `auto_update_cli` | `true` | Оба файла отсутствуют или ключ не задан |
| `serena.mode` | `stdio` | Оба файла отсутствуют или ключ не задан |
| `telemetry` | `false` | Оба файла отсутствуют или ключ не задан |
| `language` | `en` | Оба файла отсутствуют или ключ не задан |
| `model_preset` | `claude` | Оба файла отсутствуют или ключ не задан |
| `translation_voice` | `balanced` | Оба файла отсутствуют или ключ не задан |
| `timezone` | Системный часовой пояс | Оба файла отсутствуют или ключ не задан |

## Обоснование порядка чтения

Конфигурация проекта читается первой, потому что она представляет более специфичный контекст — репозиторий, в котором разработчик активно работает. Команда может задать `language: ko` или `model_preset: mixed` для своего проекта, и эти выборы не должны молча перекрываться личным глобальным `oma-config.yaml`.

Глобальный файл задаёт базовую линию на уровне пользователя. Ключи, которые проект не задаёт, наследуются из глобального значения, которое, в свою очередь, наследуется из жёстко закодированного значения по умолчанию.

## Примечания

- `language` в `oma-config.yaml` управляет языком ответов агентов. Он **не** используется для определения предупреждающих сообщений установки/обновления — те опираются на системную локаль (`$LANG`), поскольку `oma-config.yaml` ещё не загружен на момент установки.
- Приоритет `auto_update_cli` явно реализован в команде update. Когда одновременно присутствуют установка для проекта и глобальная установка, сначала проверяется `oma-config.yaml` проекта.
- Редактировать `oma-config.yaml` напрямую безопасно. `oma install` и `oma update` используют замену полей на уровне regex и сохраняют отредактированные пользователем ключи, которыми они не управляют (например, пользовательские переопределения `agents:`, `session.quota_cap`).
